# VIGIL Analyst (iOS)

On-call escalation app for VIGIL staff (`vigil_analyst`, `vigil_admin`).

**Pure React Native 0.74 + Xcode. No Expo, no expo-*, no EAS, no Expo Router.**

## Prerequisites

- macOS (required for iOS builds)
- Xcode 15+ with Command Line Tools installed
- Node.js 18+
- Ruby 2.7+ (for CocoaPods)
- An Apple Developer account is **only** required to deploy to a physical
  device or to TestFlight/App Store. The simulator works without one.

## Installation

```bash
cd mobile
npm install
cd ios && pod install && cd ..
```

## Run on iOS simulator

```bash
npm start          # in one terminal
npm run ios        # in another
```

Or open `ios/VIGILAnalyst.xcworkspace` in Xcode and hit ⌘R.

## Run on a physical device

1. Open `ios/VIGILAnalyst.xcworkspace` in Xcode.
2. Under **Signing & Capabilities**, select your team. This is where the
   Apple Developer account is required — without one Xcode cannot
   provision the bundle for a physical device.
3. Select your device in the toolbar and press ⌘R.

## API base URL

Default points at `http://localhost:8000` (the api service). On a
physical device or simulator you'll need to point at your running
backend — change it in **Settings** inside the app, or set the
`@vigil/base_url` AsyncStorage key directly.

## Authentication

Tokens live in the iOS Keychain
(`kSecAttrAccessibleWhenUnlockedThisDeviceOnly`). The app auto-refreshes
expired access tokens via `/auth/refresh`; if refresh fails the user is
returned to the login screen.

## App Store deployment

1. Bump the version in `ios/VIGILAnalyst/Info.plist` and `app.json`.
2. **Product → Archive** in Xcode.
3. Distribute via App Store Connect → TestFlight → Submit for review.

A bundle identifier reservation in App Store Connect is required before
the first archive upload. Use a reverse-DNS scheme like
`com.<yourorg>.vigil.analyst`.

## Notes

- The `ios/` Xcode project is generated by `react-native init` (or the
  community CLI). This repository checks in only the JS/TS sources;
  run the init step once locally to scaffold the native project, then
  copy `App.tsx`, `index.js`, `app.json`, and `src/` over.
