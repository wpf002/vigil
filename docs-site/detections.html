Detections

YAML is the source of truth. Compiled artifacts under detections/compiled/ are auto-generated — never hand-edit.

Anatomy of a detection

id: D1
name: Brute force credential access
attack:
  tactic: credential-access
  technique_id: T1110
state_impact:
  phase: credential-access
  status: Suspected
  confidence_contribution: 0.4
detection_logic:
  type: threshold
  field: user
  count: 5
  window: 5m
fields:
  - source.ip
  - user.name
  - event.outcome

CLI usage

# Validate every YAML file in detections/yaml/
python -m signal_translation.cli validate

# Compile to all backends + write the manifest
python -m signal_translation.cli compile

# Generate ATT&CK coverage report
python -m signal_translation.cli coverage

Detection lifecycle

  1. Branch and edit YAML under detections/yaml/.
  2. Open a PR — CI validates schema, ATT&CK mapping, and recompiles.
  3. On merge, detections/compiled/manifest.json updates and the detection-engine seeds a new active version on next startup.
  4. Per-tenant overrides ride on top — a tenant can ship an override that wins over the platform version, then roll back.

Marketplace

Tenants can publish their detections to a shared marketplace and import others. VIGIL staff publish under is_curated=true; tenants browse, filter by ATT&CK tactic, and import with one click. Imports are deduplicated per (tenant, listing) and increment a download counter for the publisher.
