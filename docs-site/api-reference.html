API Reference

All public REST endpoints across the VIGIL services.

Authentication

VIGIL accepts two bearer-token forms on every endpoint:

  • JWT — issued by POST /auth/login. Used by the web app and mobile app.
  • API key — generated under Settings → API Keys, format vgl_<48-hex>. Used by integrations and the SDK. Subject to scope checks.
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
Authorization: Bearer vgl_a1b2c3d4...

Error envelope

{ "error": "Bad Request", "detail": "Password must be at least 12 characters" }
