VIGIL Platform

AI-native, SIEM-agnostic security operations. Replaces alert queues with persistent, deterministic AttackState narratives.

What VIGIL is. A correlation, narrative, and response layer that sits on top of your existing SIEM (Splunk, Sentinel, or Elastic). Detections live as YAML, compile to SPL/KQL/EQL, and feed an attack-state engine that scores progression rather than alerts.

How it works

  1. Ingest. Pull Notable Events / Sentinel Incidents / Elastic alerts; normalize to a Common Data Model.
  2. Correlate. Map signals to AttackState objects via the entity graph and YAML detection registry.
  3. Narrate. Claude generates the attack story; deterministic confidence drives escalation.
  4. Respond. Temporal-backed playbooks dispatch to XSOAR / Tines / your stub.
  5. Report. Compliance evidence packs (SOC 2, PCI, NIST) pull from live data.
